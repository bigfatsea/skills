# 封面设计工具卡：输入来源、提示词写法、规格与出图

**模块 C（封面设计）**的方法论引擎——可独立运行（仅封面入口），也可挂在专辑或单曲之下。封面是视觉层的 Prosody（见 SKILL 核心原则）：主体意象、色彩、排版、构图都服从与音乐相同的情感意图。细则见 guide §36；本文件存**三种输入来源的映射方法、提示词写法、产出规格、以及怎么实际出图**。

## 三种输入来源，同一套映射目标

不论输入来自哪里，模块 C 始终把输入映射到同一套四要素——**主体意象 / 色彩 / 排版 / 构图**：

- **专辑语境**（完整专辑入口，缝合定稿之后；或仅封面入口，接到一份已有的 `album-blueprint-*.md` / 已完成的专辑文件）：四要素取自专辑的**throughline 意象 + 最强专属锚点 + 情感基调 + 风格家族 + Persona 气质**——此时专辑名、throughline、Persona、风格家族均已锁定。
- **单曲语境**（仅封面入口，接到一份 `song-*.md` 产出；或完整专辑里某曲需要单独出一版单曲封面）：四要素取自**这首歌自己的意象 + 它的 Suno 曲风 + 歌词的情感register**——没有专辑级 throughline 可依托，所以由这首歌自己最强的锚点句/意象承担构图。
- **自由语境**（仅封面入口，只给了一段故事或风格简述，没有先前的蓝图或单曲产出）：直接从这段简述里提炼四要素，方法与 Step 1 素材蒸馏同源、但更轻——只需定位一个具体的主体锚点、一个情感基调、一个风格家族，不需要产出完整的素材蒸馏文件。

不论来源，都要**一次性给出 3 种真正分叉的风格**（非换皮，呼应模块 B / 模块 S 的三草图精神）：**具象摄影 / 插画手绘 / 抽象极简（或拼贴）是默认原型、按主体气质可替换**，或同一主体走不同色调与排版；只须三案真正分叉。每种写：①视觉概念一句话；②四要素取值；③可直接喂图像模型的提示词。

- **主体 = throughline 意象 / 最强专属锚点**（专辑或单曲语境）或**简述自身的具体锚点**（自由语境）——具体物件优先、sense-bound，别用抽象泛图；throughline 不可视觉化时退到锚点物件。构图**留出叠标题的负空间**，缩略图（流媒体首图极小）要焦点强、信息少。
- **真人 / 在世主角**：默认不依赖可辨识人脸——用 throughline 实物、手/背影等局部或插画（既规避肖像授权，也因 AI 本就画不出特定真人）；确需本人肖像须家人授权。敏感意象按 guide §27 抽象化。

## 提示词写法

- **提示词两套写法**：Midjourney 用"主体 + 风格/媒介 + 光线 + 色彩情绪 + 构图"并加 `--ar 1:1`（≤约 60 词）；Nano Banana / GPT Image 用自然语言句（Subject + Action + Scene + mood）。**提示词统一加 `no text`/"do not render any text"**，防模型自己糊假字母占掉留白。
- **AI 文字渲染不可靠**：专辑名 / 主角名优先**后期叠字**，或用文字渲染强的模型（Nano Banana Pro / GPT Image）；别指望 Midjourney 把标题拼对。

## 规格

- 流媒体母版 **3000×3000、1:1、sRGB、JPG/PNG**；若要实体 CD，另出 **126×126mm（含 3mm 出血）、300 DPI（≥1417px）、CMYK**、文字留安全区。AI 先出最高分辨率方形再放大到 3000²。

## 实际出图（Production，非纯 Design）—— 用 `ai-script`

纯 Design 运行到提示词为止。要实际出图，用 `ai-script` CLI 的 `image` 子命令（本机已装好并配置，完整参考见 `ai-script` 技能）；本 skill 不再自带出图脚本——`ai-script image` 已覆盖同样的图像服务商（默认 `grsai:gpt-image-2`，`--list` 可看 Google/OpenAI/Nano Banana 等其他选项）。

**`ai-script` 必须从它自己的仓库目录里调用**（其 wrapper 用 `uv run`，靠当前工作目录定位项目）——**不是**专辑的工作目录。用**绝对路径**指定输出位置，让图片落到专辑工作目录内，而不是落进 `ai-script` 自己的仓库：

```
cd ~/code/ai-script && ./ai-script image "<提示词>" -o {工作目录绝对路径}/covers/{名称}-{风格}.png -e grsai:gpt-image-2
```

- **提示词较长/含引号换行时用 `--stdin`**（封面提示词常常如此）：`echo "<提示词>" | ./ai-script image --stdin -o {绝对路径}/covers/{名称}-{风格}.png`。
- **先检查环境**：`./ai-script check --no-network` 校验所选服务商的密钥是否就绪；别为了确认密钥缺失而先跑一次昂贵的 `image` 调用。
- **超时**：`grsai` 出图通常 20–60 秒，但高峰期观测到过约 511 秒——Bash 超时设 ≥ 600 秒，高峰期加 `--retries 2`。
- **3 版封面 = 3 次独立调用**（每种风格一次）；批量场景 `ai-script image` 也支持 `--batch` 配 `tasks.jsonl`（见 `ai-script guide image`）。
- **所有产出图片放目标项目（工作）目录下的 `covers/` 子目录，不要散落到别处。**

**交互模式**可用 `AskUserQuestion` 让用户选定 1 种主封面（其余留作备选）；**自动模式**选最贴主体气质的一种并说明理由。
