# 单曲创作工具卡：独立入口契约 + 逐曲创作工具卡

**模块 S（单曲创作）**的方法论引擎——本 skill 最核心的一项能力，也是唯一能仅凭一小段 brief 独立运行的能力。它把 guide §14（重写）与 guide §15（七维模型）落成可执行步骤。本文所有 `§N` 引用均指 `references/album-creation-guide.zh.md`。

模块 S 运行在**两种语境**下，共用下面同一套创作工具卡，但产出规模的默认值不同。**用哪种语境，取决于这首歌是否已经有模块 B 下发的曲目档案——而不是用户这次敲的是"仅单曲"还是"完整专辑"这个入口名。** 一次仅单曲请求既可能落进独立语境，也可能落进专辑语境（见 SKILL Step 0/4）：

- **独立语境（这首歌没有已有蓝图）**：输入是一小段 brief 或故事片段，没有专辑框架——不论是通过独立的仅单曲请求到达，还是任何其他"这首歌没有模块 B 档案"的情形。默认产出 = **3 份取向不同的歌词，每份配一套一一对应的 Suno 曲风，外加 3 个候选歌名**。见下文"独立入口契约"。
- **专辑语境（已存在曲目档案——来自完整专辑跑模块 B 的派发，或来自仅单曲请求接续此前"仅专辑蓝图"/完整专辑跑出的某一首具体曲目）**：输入是 SKILL Step 3 冻结的创作契约卡（全专宪法）加本曲档案。产出数量遵循**分层投放表**（不变，见下文"专辑语境分层投放"）——因为专辑内部已经按角色（主打/副主打/底座/间奏）做了区分；独立单曲没有这样的角色可供区分，所以默认对 3 份取向全部满格投入。

这两套契约并不矛盾——只是同一套创作工具卡在两种不同尺度上的应用。**不要把专辑的分档折扣套用到真正独立的单曲上，也不要把独立单曲"3 份满格"的默认值套用到已经在蓝图里有定位的曲目上**（一首通过仅单曲一首一首接续出来的底座曲依然是底座曲——给它 3 份满格取向会无理由地推高专辑的成本预算）。

## 独立入口契约（没有已有蓝图）

**输入**：一小段 brief 或故事片段（一段文字、一段记忆、一个场景——不需要完整素材包；若用户确实给了更完整的素材包，先把它读全，同 guide §5"sense-bound 取材"的精神），加上 SKILL Step 0 确认的最小 brief（语言、是否真实人物、大致情绪方向）。不需要素材蒸馏文件、不需要专辑三方向 gate、不需要蓝图、不需要契约卡——那些是模块 B 的工作，在此不是前提条件。**若这首歌已经有蓝图和曲目档案，本节不适用——改用下文"专辑语境分层投放"，即便用户这次是从仅单曲入口进来的。**

**任务**：一次性产出——

1. **3 个候选歌名**（不与 3 份歌词一一绑定——先给出能配任意一份的候选歌名，等各版歌词写出来后再确定推荐配对）。
2. **3 份真正分叉的歌词**，用与模块 B 三草图相同的分叉方法（`references/blueprint-kit.zh.md`"同一套分叉方法，落到单曲粒度"）但缩到单曲尺度：至少选一个轴（故事内核 / 情感基调 / 风格取向）让它真正分叉，而不是同一版本换皮（例如同一份素材，可以给"直白叙事+民谣"「克制留白+抒情 soul」"意象化+合成器流行"三种取向）。每份都要跑完下文的**完整三轮打磨协议**——3 份都不打折，因为没有专辑角色可供分级。
3. **3 套一一对应的 Suno 曲风**，每份歌词配一套——曲风要从这份歌词自身的情感register与意象里推导，而不是三份共用一套曲风描述。写成"克制留白+抒情 soul"取向的歌词，不能配一套读起来像合成器流行取向的曲风卡。
4. **3 份都做额外锤炼**（不只是指定某一份为"主打"）：下文的金句双轨对打和好听三测试，对全部 3 份都适用——因为在一首独立单曲里，每一份都是"这首歌"的候选，不是只需过门槛的底座曲。
5. **自评并推荐一份**，附简短择优理由 + 简短倒字/QC 备注——与下文专辑语境的 brief 相同，但这里的推荐只是给用户参考、供其与另外两份权衡，不是不可逆的收窄（3 份都留在最终交付物里）。

**执行方式**：一首独立单曲不会遭遇专辑语境里"一条长上下文背 8 首歌"的注意力稀释问题——但把 3 份真正分叉的取向在同一个上下文里背靠背写完，会有相反的风险：各版本互相靠拢、分叉被磨平。**为每份取向派一个独立 subagent**（3 个并行），每个都明确分到自己的分叉角度（如"你负责克制留白版"），并附上本文件 + 用户的 brief；随后由主上下文对 3 份统一跑一遍裸读 QC（见 SKILL 核心原则），再呈现给用户。这和专辑语境"每曲一个 subagent"的做法是同一种机制，只是把粒度从"曲目"换成了"取向"。

**产出命名**：`song-{歌名}-{model name}.md`，包含 3 份取向（歌词 + 曲风 + 歌名 + 三轮打磨记录 + 金句双轨对打 + 每份的好听三测试备注）、推荐理由、以及每份的简短 QC 备注（倒字自检、裸读检查）。不再拆一份单独的"候选与打磨"文件——对一首单曲而言，候选本身就是交付物。

## 专辑语境分层投放（不变）

**发给每个逐曲 subagent 的 brief**（由 SKILL Step 4 派发——不论是完整专辑一次性对全部曲目 fan-out，还是仅单曲请求接续蓝图里的这一首）：

- **输入** = 创作契约卡（全专宪法）+ 本曲档案（叙事语境/锚点/金句/杠杆/throughline 触点）+ 本文件 + `references/album-creation-guide.zh.md` 路径备查。**subagent 不读完整素材，但必须知道这首歌背后是什么故事**——本曲档案的"叙事语境"和"与本曲相关金句"字段是它的情感锚地，主上下文须认真填写，而非只给锚点符号。
- **任务** = 产出 ①3 个候选歌名；②按下表分层的若干版完整歌词（每版过"三轮打磨协议"；歌词语言默认普通话，若本曲叙事节点有显著英语文化特征可提出中英混语或全英文建议，附理由供确认）；③Suno Style 曲风；④自评择优选出推荐版，附简短择优理由 + 倒字/QC 简注。
- **约束** = 服从契约卡；不得自创 Persona/风格家族/throughline；遵守本 SKILL"硬性写作纪律"。

| 曲位 | 歌词版本 | 三轮打磨 | 曲风(Style) | 目标 | 生产 SOP |
|---|---|---|---|---|---|
| **主打** | **1 完整 + 金句双轨 + 2 候选副歌** | 完整版 3 轮 + 金句锤炼 | **3 种（含人声风格）** | 85+ | N≥8、定 Persona、副歌 Inpainting 2–5 次、逐句校倒字（guide §31） |
| **副主打** | 2 版 | 每版 3 轮 + 好听三测试 | **2–3 种（侧重旋律/曲风抓耳）** | 80+（好听优先） | **高 N 抽卡求旋律命中**、Inpainting 调编曲、词守门槛 |
| **底座** | 标准档 3 版 / 经济档 1 版 | 每版 3 轮 | **1 种家族内**（可+1 备） | 60+ | N=2–4、复用 Persona+风格家族、合格即收 |
| **器乐间奏** | — | — | 1 种（开 Instrumental，Lyrics 留空）+ 结构说明 | — | 多抽卡取最暖一条 |

> **底座不做 3 曲风**：底座曲风已被风格家族+Persona 锁死，强行 3 种要么破坏专辑一致、要么是化妆式微调（浪费）。**曲风不是底座的变量，歌词才是。**

## 每版歌词/每份取向，须逐项写全（七维落地）

1. **维度 1 词与叙事**：素材锚点 → 意象化（以具体喻情，不直白诉苦）→ 副歌 hook（短、可重复、专属意象+普世情感）→ 视角（§6）→ 结构落点（私密细节进主歌、深情留白进副歌）。
2. **维度 2 情感→杠杆**（§3/§18）：定调式、速度/拍号、配器、动态、人声，并写明每个取值"为什么"服务情感。情感弧（如小调主歌→大调副歌）落到调式+动态的段内变化。
3. **维度 3–7**：曲风 / 曲式与动态弧 / 节奏 / 人声 / 乐器配器。
4. **Suno Style 字段**（§21）：4–7 个描述符；**人声三层（音色+唱法+制作）放最前**，可加**调性(key)**提升一致性，核心乐器点名（民乐用 erhu/guzheng 等具体名）。音色交给 Style+Persona，演唱弧与配器变化下放 metatag——**两个控制层级**（完整 tag 列表见 §22C）：① **段落冒号语法** `[Chorus: full band, erhu accent, powerful vocals]` — 覆盖该段整体制作+演唱风格，叠加（而非替换）全局 Style；② **行级 delivery tag** `[Whispered]`/`[Belted]` 独占一行、紧贴下方歌词行——仅作用于该行、仅限人声层。附 Sliders（情感歌=Weirdness Safe + Style Influence Strong + Prompt Helper Off）。
5. **完整中文歌词**（见 SKILL 的"硬性写作纪律"）：副歌放最强锚点、句内气口用标点/换行、段落按需不套满。
6. **中文可唱性 / 倒字自检**（§13/§22）：逐一检查 hook 关键词与人名零倒字，标出需重点盯的长句。

## 三轮打磨协议（不分独立/专辑、不分底座，全员执行）

即 guide §14 的三个阶段，落成可执行步骤。三轮**必须按序、绝不一次到位**（一起做正是句子僵硬绕口的成因）。每版歌词跑完三轮，并**留一段简短打磨记录**（哪轮改了什么）：

```
第1轮 叙事先行(规则全关)：像讲故事一样把这首歌的情感真相与场景白描出来，把故事和专属锚点弄对。
                 圈出本节点可命中的逐字金句——不必全用；最戳的一两句通常就够(按最感官+最短可唱+最普世选)，只有确实各得其所才多留。此轮无视押韵/格律/工整。
第2轮 歌词化(prosodic shaping)：段落职能(主歌=铺陈/show、副歌=记忆点/hook)、锁一辙一韵到底、
                 【副歌密铺对仗·主歌按需埋对句】、行长齐(≤10–12字)、砍口语化的散句（如"你们都知道""我自己扛"）、
                 压缩 hook、稳定/不稳定工具按情感取用、副歌逐次写全
第3轮 精修+把关(逐字)：先裸读(当作不知道你意图的母语听者冷读——每一行、每对相邻行都要读得通：
                 问句与答落在同一维度、无句式只开不合、无一词两义打架、无残句悬空；此关先于一切形式打磨，任何锁辙/换字都是再创作、须重读)，
                 再做最强锚点上副歌信号位、倒字自检(题眼/人名零倒字)、高音/长音落开口韵、
                 气口标点化(禁裸空格)、metatag 双锚、跑删形容词/留白/跟唱三测试
```

> 这套协议只花文本 token，是修复词作质量的最高 ROI 单点。三轮记录连同各版歌词一并保留——专辑语境写入 Step 5"候选与打磨"附录；独立语境直接留在 `song-*.md` 交付物内。

## 额外锤炼（专辑主打/副主打——必做；独立单曲——3 份全做）

在三轮打磨之外：

- **金句双轨对打**：核心 hook 同时产出**「极简留白版」与「丰满工整版」**两条候选正面对打、择优——**强制保住张力，防被单向工整化稀释**（这正是修复"好句被押韵磨钝"的开关）。**三测之前，killer line 必须先过裸读**——它的劲道全靠这行字读得通，若是问答，答必须落在问题的同一维度（问"疼不疼"用"我非常满意"这类满意度评价作答即不通——要答*疼*这个维度："不疼了"/"值了"/"无怨无悔"）。再择优过三测：**删形容词测试**（删了会不会更狠）、**留白测试**（有没有替听众把话说完）、**跟唱测试**（听一遍能不能跟着唱出来）。
- **好听三测试**：朗朗上口测试 / 跟唱测试 / **旋律记忆点测试（hook 能不能哼出来）**。专辑语境下副主打以此为首要验收口径；独立语境下 3 份都以此验收。

> 边界：**金句留白只用在锚点的 1–2 句上，不全曲推行**——全曲留白会让歌词变稀、反伤好听。
