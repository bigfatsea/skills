# 逐曲词作工具卡

Step 4 发给每个逐曲 subagent 的创作工具卡，**与创作契约卡、本曲档案一并交付**。它把框架 §14（重写）与 §15（七维模型）落成可执行步骤。subagent 服从契约卡与本 SKILL 的"硬性写作纪律"；需深查引擎时再按路径读 `references/album-creation-guide.zh.md` 的相关 §。

## 每版歌词，subagent 必须逐项写全（七维落地）

1. **维度 1 词与叙事**：素材锚点 → 意象化（以具体喻情，不直白诉苦）→ 副歌 hook（短、可重复、专属意象+普世情感）→ 视角（§6）→ 结构落点（私密细节进主歌、深情留白进副歌）。
2. **维度 2 情感→杠杆**（§3/§18）：定调式、速度/拍号、配器、动态、人声，并写明每个取值"为什么"服务情感。情感弧（如小调主歌→大调副歌）落到调式+动态的段内变化。
3. **维度 3–7**：曲风 / 曲式与动态弧 / 节奏 / 人声 / 乐器配器。
4. **Suno Style 字段**（§21）：4–7 个描述符；**人声三层（音色+唱法+制作）放最前**，可加**调性(key)**提升一致性，核心乐器点名（民乐用 erhu/guzheng 等具体名）。音色交给 Style+Persona，人声段落弧下放 metatag。附 Sliders（情感歌=Weirdness Safe + Style Influence Strong + Prompt Helper Off）。
5. **完整中文歌词**（见 SKILL 的"硬性写作纪律"）：副歌放最强锚点、句内气口用标点/换行、段落按需不套满。
6. **中文可唱性 / 倒字自检**（§13/§22）：逐一检查 hook 关键词与人名零倒字，标出需重点盯的长句。

## 三轮打磨协议（全员执行，含底座）

即框架 §14 的三个阶段，落成可执行步骤。三轮**必须按序、绝不一次到位**（一起做正是句子僵硬绕口的成因）。每版歌词跑完三轮，并**留一段简短打磨记录**（哪轮改了什么）：

```
第1轮 叙事先行(规则全关)：像讲故事一样把这首歌的情感真相与场景白描出来，把故事和专属锚点弄对。
                 圈出本节点可命中的逐字金句——不必全用；最戳的一两句通常就够(按最感官+最短可唱+最普世选)，只有确实各得其所才多留。此轮无视押韵/格律/工整。
第2轮 歌词化(prosodic shaping)：段落职能(主歌=铺陈/show、副歌=记忆点/hook)、锁一辙一韵到底、
                 【副歌密铺对仗·主歌按需埋对句】、行长齐(≤10–12字)、砍口语化的散句（如"你们都知道""我自己扛"）、
                 压缩 hook、稳定/不稳定工具按情感取用、副歌逐次写全
第3轮 精修+把关(逐字)：先裸读(当作不知道你意图的母语听者冷读——每一行、每对相邻行都要读得通：
                 问句与答落在同一维度、无句式只开不合、无一词两义打架、无残句悬空；此关先于一切形式打磨，任何锁辙/换字都是再创作、须重读)，
                 再做最强锚点上副歌信号位、倒字自检(题眼/人名零倒字)、高音/长音落开口韵、
                 气口标点化(禁裸空格)、metatag 双锚、跑删形容词/留白/跟唱三测试
```

> 这套协议只花文本 token，是修复词作质量的最高 ROI 单点。三轮记录连同各版歌词一并保留，供 Step 5 写入"候选与打磨"附录。

## 主打 / 副主打额外锤炼（成本只是多几行文本）

两首主打的 subagent 在三轮打磨之外，再各做一道：

- **金句双轨对打（仅主打）**：核心 hook 同时产出**「极简留白版」与「丰满工整版」**两条候选正面对打、择优——**强制保住张力，防被单向工整化稀释**（这正是修复"好句被押韵磨钝"的开关）。**三测之前，killer line 必须先过裸读**——它的劲道全靠这行字读得通，若是问答，答必须落在问题的同一维度（问"疼不疼"用"我非常满意"这类满意度评价作答即不通——要答*疼*这个维度："不疼了"/"值了"/"无怨无悔"）。再择优过三测：**删形容词测试**（删了会不会更狠）、**留白测试**（有没有替听众把话说完）、**跟唱测试**（听一遍能不能跟着唱出来）。
- **好听三测试（主打 + 副主打）**：朗朗上口测试 / 跟唱测试 / **旋律记忆点测试（hook 能不能哼出来）**。副主打以此为首要验收口径。

> 边界：**金句留白只用在主打的 1–2 句锚点上，不全曲推行**——全曲留白会让歌词变稀、反伤好听。
