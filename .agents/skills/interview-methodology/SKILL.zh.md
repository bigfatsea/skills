# 访谈方法论

贯穿访谈全流程的三模式技能：**准备 → 整理 → 提炼**。AI 是生产辅助工具；真实访谈始终由人类采访员主导。

```
模式一 访前准备  →  模式二 转写整理  →  模式三 故事提炼
     ↓                   ↓                    ↓
研究报告              结构化笔记           叙事内核
+ 访谈提纲            + 言外之意日志        → 输入 audio-album-creator
```

---

## 核心原则

- **AI 辅助，人类主访。** 建立信任、实时沉默管理、共情——这些都是无法参数化的默会知识（Polanyi）。AI 的职责：准备、处理、识别模式——绝不替代现场访谈。
- **叙述者的主观真相，而非客观事实。** 记录矛盾，不调和矛盾。（Portelli：口述史揭示的是事件的「意义」，而非事件本身。）
- **永远不要直接信任 AI 转写。** 中文 ASR 词错误率通常为 5–15%；方言/老年人/专业词汇场景可达 30%+。每份转写稿都需要对照原音频进行人工核验。
- **访谈时长：60–90 分钟。** 超过 90 分钟，叙述者疲劳会导致信息密度下降。开始前应与受访者约定时长。
- **场景重于概述。** 一个具体细节胜过一百个形容词。
- **伦理优先，始终如此。** 知情同意先于任何录制。叙述者保有退出权、化名权和访问限制权。

## 模式一 — 访前准备

**触发条件：** 受访者已确定；访谈尚未进行。

**输入：** 受访者姓名/化名 · 项目目标 · 已知背景（可选）

**输出：**
1. 研究维度报告 — 8 维度表格（`references/ethics-and-prep.zh.md §B.3`）
2. 半结构化访谈提纲 — 话题组 + 每组 3–5 个开放性问题 + `[目的：]` 注释 + 5 个备用问题；最多 2 页 A4
3. 伦理前置核查单 — 同意书提醒、敏感领域标记

输出文件：`pre-interview-{subject}.md`

**Agent prompt:**
```
任务：生成访前研究报告和半结构化访谈提纲。

输入：受访者姓名/化名 + 项目目标 + 已知背景

输出：
1. 研究维度报告（填写 8 维度表格，references/ethics-and-prep.zh.md §B.3）
2. 半结构化提纲：话题组 + 每组 3–5 个开放性问题 + [目的：] 注释 + 5 个备用问题；最多 2 页 A4
3. 伦理前置核查单：标记敏感领域，提示同意书

约束：从话题组入手，而非从预设问题入手。避免引导性问题。在访谈前标记法律/伦理敏感领域。
```

---

## 模式二 — 转写整理

**触发条件：** 访谈已完成；原始 AI 转写稿需要整理。

**输入：** 原始转写稿 · 原始音频文件（核验必需）· 受访者背景 · 访谈类型（人生 / 项目 / 梦境 / 事件 / 关系）

**输出：**
1. 纠错转写稿 — 三轮 SOP 执行完毕；段落标注 `[已核验]` / `[存疑]` / `[听不清]`
2. 结构化笔记 — 用对应类型模板填写（`references/transcript-templates.zh.md §E.3.x`）
3. 言外之意日志 — 非语言信号、停顿、回避、情感变化、矛盾
4. 追访问题清单 — 缺口与未解问题

输出文件：`transcript-{subject}-{date}.md` · `structured-notes-{subject}.md`

**Agent prompt:**
```
任务：对访谈转写稿进行纠错和结构化整理。

输入：原始转写稿 + 音频文件 + 受访者背景 + 访谈类型

输出：
1. 纠错转写稿（三轮 SOP，references/transcript-templates.zh.md §E.1；标注 [已核验]/[存疑]/[听不清]）
2. 结构化笔记（E.3.x 模板，按访谈类型选择）
3. 言外之意日志（停顿、回避、重复、情感强度变化、矛盾、身体语言）
4. 追访问题清单

约束：忠于叙述者的表达方式——不美化其语言。标注矛盾，不调和矛盾。言外之意日志独立成节，不嵌入正文。核验所有存疑段落时必须对照原始音频——绝不接受未经核验的 AI 转写。
```

## 模式三 — 故事内核提炼

**触发条件：** 结构化笔记已存在；需要叙事内核和角度推荐用于创作。

**输入：** 结构化笔记 · 受访者背景 · 项目目标

**输出：**
1. 主题识别 — 五法表（频次 / 情感强度 / 反差 / 二元对立 / 省略）
2. 故事弧 — 建置 / 冲突 / 解决 + 转折点、低谷、顿悟
3. 六角度评估 — 奋斗成长 / 关系羁绊 / 冲突对抗 / 时代镜像 / 逆转顿悟 / 坚守放弃（适用性 + 每角度一句话概述）
4. 推荐角度 — 理由：叙述者情感峰值 + 最强场景素材 + 项目目标契合度
5. 关键场景清单 — 具体、感官、可重现的特定时刻
6. 一句话故事测试：`[人物]在[情境]中，经历了[转折]，最终[变化/选择]`
7. 五问故事测试 — 唯一性 / 共鸣性 / 值得讲述 / 可信度 / 完整性

输出文件：`story-core-{subject}.md`

**Agent prompt:**
```
任务：从访谈整理笔记中提炼故事内核，推荐叙述角度。

输入：结构化笔记 + 受访者背景 + 项目目标

输出：
1. 主题识别（五法表：频次 / 情感强度 / 反差 / 二元对立 / 省略）
2. 故事弧（建置 / 冲突 / 解决；转折点、低谷、顿悟）
3. 六角度表（奋斗成长 / 关系羁绊 / 冲突对抗 / 时代镜像 / 逆转顿悟 / 坚守放弃 — 适用性 + 一句话概述）
4. 推荐角度 + 理由（叙述者情感峰值 + 最强场景素材 + 项目目标契合度）
5. 关键场景清单（具体、感官、尽可能带时间戳）
6. 推荐角度的一句话故事测试
7. 五问故事测试结论（唯一性 / 共鸣性 / 值得讲述 / 可信度 / 完整性）

约束：让素材自己浮现角度——不强加框架。情感强度 > 叙事整洁度。每个角度标注：适用 / 边缘 / 不适用（附依据）。
```

## 伦理 — 始终适用

| 步骤 | 动作 | 时机 |
|---|---|---|
| 1 | 准备项目说明 + 录音同意书 + 版权授权书 | 首次联系前 |
| 2 | 访前会面中逐条解读文件；口头确认 | 访前会面 |
| 3 | 录音中记录口头同意（on-record） | 每次录制开头 |
| 4 | 签署版权授权书（权利转让、发布范围） | 访谈完成后 |

每次录制前执行完整伦理自查（`references/ethics-and-prep.zh.md §A.6`）。

---

## 下游对接 → audio-album-creator

- **对接步骤一**（素材摄入与提炼）：`structured-notes-{subject}.md` + `story-core-{subject}.md` 替代原始音频/文本，album creator 收到预处理素材，减少步骤一工作量并提升锚点精度
- **对接步骤二**（三个创意方向）：六角度评估直接映射为三个方向草案；推荐角度作为「推荐」草案的种子
- **对接步骤三**（专辑蓝图）：一句话故事测试成为蓝图概念；关键场景清单填充感官锚点池

将 `story-core-{subject}.md` 放入步骤零的素材目录。Album creator 无需重读原始转写稿。
