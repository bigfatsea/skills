使用此清单来决定是否需要添加比较表的行。

仅在选项会实质性地影响以下方面时，才创建决策行：

- 架构简洁性
- 交付速度
- 实现风险
- 运行时可靠性
- 用户体验质量
- 性能
- 部署/打包复杂性
- 长期可维护性
- 平台兼容性或权限模型
- 系统状态恢复或还原行为

表明需要决策行的强烈信号：

- 用户已指定技术栈，但仍存在多条可行的实现路径
- 功能跨越操作系统、浏览器、设备、桌面或外部 API 边界
- 取巧的快捷方式会产生在实际使用中表现不佳的演示
- 产品需要具体的默认值以实现开箱即用
- 某个选项能更好地处理故障模式、清理或状态恢复
- 打包、分发、签名、安装或权限授权并非简单事项

不要为琐碎的实现细节创建行。

在比较替代方案时：

- 首先优先考虑原生或平台默认解决方案
- 当质量相当时，优先选择依赖项更少的方案
- 优先选择确定性行为，而非巧妙花招
- 优先选择具有更清晰故障处理和恢复路径的解决方案
- 明确指出那些诱人但质量低劣的捷径
- 包括权衡取舍，而不仅仅是建议
