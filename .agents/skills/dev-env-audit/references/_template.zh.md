## 防幻觉自检清单（新增/修改 reference 时逐条过，不可跳过）

LLM 写这类文档最典型的错误是**把两个真实事物的记忆拼接成一个不存在的东西**
（真实教训：`asdf set --global` = 旧版 `asdf global` 拼上新版 `asdf set`；
`UV_VIRTUALENV_DIR` = uv 真实变量的命名风格拼上并不存在的全局 venv 概念）。
凭记忆写完之后，逐条核实：

- [ ] **每个命令/子命令/flag**：在本机真跑一遍，或至少 `<tool> <subcmd> --help` 确认该 flag 存在。工具跨大版本改过 CLI 的（asdf 0.16 重写）要注明适用版本。
- [ ] **每个环境变量**：官方文档确认；本机装了该工具时可用 `strings "$(command -v <tool>)" | grep <VAR>` 验证二进制里真有这个名字。
- [ ] **每个安装渠道**：`brew info <formula|cask>` 确认存在且分清 formula/cask；安装脚本 URL 与官方 README 逐字核对，不凭记忆默写域名。
- [ ] **每个"某某工具默认目录/默认行为"论断**：查当前版本文档——默认值最容易随版本漂移（fnm 的默认目录就换过）。
- [ ] **版本号相关论断**（如"Apple git 冻结在 2.39.x"）：写明这是撰写时点的事实，避免绝对化表述。
- [ ] **新变量要同步进 `scripts/probe-cache.sh`**（§4 外置变量清单与脚本核查清单必须一致，别造成"reference 推荐了、横切检查看不见"的脱节）。
- [ ] 无法在本机验证的条目（工具没装）：在行内标注"未本机验证，出处：<链接/文档名>"，宁可标注也不硬写。
